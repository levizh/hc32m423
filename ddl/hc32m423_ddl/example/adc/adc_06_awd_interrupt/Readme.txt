平台说明
================================================================================
GCC工程，由Eclipse IDE外挂GNU-ARM Toolchain，再结合pyOCD GDB Server实现工程的编译、
链接和调试。在用Eclipse导入工程后，请将xxxx_PyOCDDebug中pyocd-gdbserver和SVD文件
设置为正确的路径；请将xxxx_PyOCDDownload中pyocd设置为正确的路径。注意，这些路径不
能包含非英文字符。


功能描述
================================================================================
本例程用TIMER2每100ms触发一次序列A动作，如果AWD0（模拟看门狗0）和AWD1各自的监测的通道
满足比较条件，则产生相应中断。例程主要展示了AWD0和AWD1中断的配置方法和基本用法。

AWD0和ADW1的通道分别为ANI9（P00）和ANI10（P01）。开发板上，只有一个电位器，并只连接到P00，
在测试时可将P01和P00短接，以方便测试。

================================================================================
使用步骤
================================================================================
1）USB转串口工具连接至USART3，并与电脑相连；用MicroUSB将开发板与电脑相连；
2）打开工程，重新编译，下载至开发板并运行；
3）缓慢旋转电位器，可看到串口调试助手显示AWD0和AWD1中断产生的情况；
4）也可用断点调试的方式来观察程序运行情况。

注意
================================================================================
请根据电位器输出电压的范围来设置AWD0和AWD1的比较区间。
